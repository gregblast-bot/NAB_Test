Features to extract
Feature,Interpretation for Engineer,"Likely ""Next Step"" Checklist"
Volatility,"""The resource usage is oscillating wildly.""",Check for load balancer flapping or retry loops.
Trend Slope,"""The resource is slowly leaking/filling up.""",Check for memory leaks or disk space logs.
Burstiness,"""A massive spike occurred suddenly.""",Check for new deployment or batch job start.
Instability,"""Current state is unlike the last hour.""",Immediate triage: check recent config changes.

Trigae mapping (refined failure types)
Failure Type,Primary Signal,Suggested Next Steps (Checklist)
CPU Issue,High avg_load or burstiness,"1. Run top -o %CPU to find the rogue process.2. Check for recent code deployments or ""hot"" functions.3. Verify if autoscaling failed to trigger."
Memory Issue,High trend_slope (Leak),"1. Check dmesg for ""Out of Memory"" (OOM) killer events.2. Inspect application logs for java.lang.OutOfMemoryError.3. Restart the service container/process."
Disk Issue,High volatility in I/O,"1. Check df -h for 100% full partitions.2. Run iostat -x to check for high disk ""wait"" times.3. Delete or rotate old log files in /var/log."
Network Issue,Sudden instability_index,"1. Check netstat for a spike in ""TIME_WAIT"" connections.2. Verify Cloud provider status (e.g., AWS Direct Connect).3. Look for a DDoS pattern or a sudden ""Retry Storm."""
